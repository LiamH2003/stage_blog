Blog 06

Eindspurt: alles afronden voor de finish

20 april – 1 mei 2026 Periode: week 11 en 12
01

Wat heb ik de voorbije twee weken gedaan?

Op 20 april startte ik met het omzetten van de biker game naar React Native Web. Dat ging niet van een leien dakje: veel errors, en bovendien moest ik een deel van de tutorialflow opnieuw opbouwen omdat ik opnieuw zonder recente commit zat. Een vervelende herhaling van de les die ik een week eerder al had geleerd. Ik had ook een overleg met Ghassen en Isabelle over de lengte van de tutorial, waarbij een A/B test als optie op tafel kwam.

Op 21 april lukte het. De biker game draaide volledig stabiel met React Native Web, ik committe onmiddellijk en schoonmaakte daarna de code van zowel de biker game als de puzzle game.

Op 22 april had ik een overleg met Arne en Ghassen over responsive opties, werd de biker game en puzzle game bijna volledig afgewerkt (zo goed als 98%) en werden de eerste stappen gezet voor een testomgeving voor het businessteam. Op 23 april was het bachelorproefdag.

Op 24 en 27 april werkte ik intensief verder aan de climber game: gameplay, basisfunctionaliteiten en een correcte folderstructuur. De tijd begint te dringen, dus ik werkte heel gefocust aan wat haalbaar is voor het einde van de stage.

Op 28 april sprak ik met Isabelle over de rebranding van de biker game en bereidde ik een meeting voor om de prioriteiten te bepalen tussen de rebranding en de verdere ontwikkeling van de climber game. Op 29 april werkte ik de volledige dag zelfstandig verder aan de climber game, ook al was ik door transportproblemen niet op kantoor aanwezig.

Op 30 april was het opnieuw bachelorproefdag. Op 1 mei paste ik de folderstructuur aan voor de climber game en maakte ik de overgang naar een Vite-gebaseerde setup, zodat alles consistent is met de rest van het project.

Skills die ik nodig had

React Native React Native Web Vite Debugging Code cleanup Zelfstandig werken Prioriteiten stellen
02

Deadlines en planning

De stage loopt af op 15 mei, wat betekent dat ik nu echt in de eindspurt zit. De biker game en puzzle game zijn vrijwel volledig afgewerkt. De climber game is het laatste grote werkpunt. De prioriteitsdiscussie tussen de rebranding van de biker game en het verder uitwerken van de climber game blijft open, maar de focus ligt momenteel op de climber.

✓ Biker game volledig werkend in React Native Web
✓ Biker game en puzzle game bijna volledig afgewerkt (98%)
✓ Code van biker en puzzle game opgeschoond
✓ Eerste stappen testomgeving voor businessteam voorbereid
✓ Climber game in playable state gebracht en verder uitgewerkt
✓ Climber game overgezet naar Vite-setup met aangepaste folderstructuur
→ Climber game volledig afronden
→ Rebranding van de biker game uitvoeren
→ Testomgeving openzetten voor het businessteam
03

Wat heb ik nieuw geleerd?

Het verschil tussen React Native en React Native Web is me deze weken echt duidelijk geworden. Ze lijken op elkaar maar gedragen zich anders, en code die in het ene werkt, werkt niet vanzelfsprekend in het andere. Dat inzicht helpt me nu veel sneller problemen inschatten.

Ik leerde ook wat het betekent om onder tijdsdruk te werken aan het einde van een project. Je kan niet alles perfect maken. Je moet keuzes maken over wat de meeste waarde heeft voor de oplevering. Dat vraagt om een andere manier van denken dan aan het begin van de stage, toen er nog alle tijd leek te zijn.

En de les van een week eerder over committen leerde ik op 20 april opnieuw op de harde manier. Maar dit keer was het de laatste keer. Ik commit nu consequent na elke betekenisvolle stap.

"Onder tijdsdruk leer je prioriteren. Niet alles hoeft perfect te zijn, het moet wel af zijn."
04

Mijn omgeving en collega's

Ik merk dat de sfeer aan het einde van een project verandert. Iedereen is wat drukker, meetings gaan sneller en er wordt verwacht dat je zelfstandiger werkt. Dat is soms wat eenzamer, maar ik merk ook dat ik er klaar voor ben. Ik weet intussen hoe het werkt bij AG en ik kan mijn eigen weg vinden.

Ghassen is ook in deze laatste weken een constante factor. Zijn feedback is direct en technisch scherp, en ik merk dat ik minder uitleg nodig heb dan aan het begin. We verstaan elkaar sneller.

De dag waarop ik door transportproblemen niet op kantoor kon zijn op 29 april was eigenlijk een bevestiging: ik kan ook volledig zelfstandig een volledige dag productief werken. Dat gevoel is fijn om mee af te ronden.

05

Wat doe ik goed en waar kan ik groeien?

Wat doe ik goed?

  • Consequent committen na elke stap
  • Onder tijdsdruk blijven focussen op wat telt
  • Zelfstandig werken zonder fysieke aanwezigheid op kantoor
  • Code opschonen en structureren na afronding van features

Waar kan ik groeien?

  • Eerder aangeven als prioriteiten niet duidelijk zijn
  • Niet wachten op assets maar alternatieven voorzien
  • Realistischer plannen aan het begin van een sprint

In de allerlaatste weken wil ik bewust blijven plannen: wat moet echt af, wat is nice to have en wat laat ik los. Dat onderscheid maken is iets wat ik aan het begin van de stage niet goed kon, maar nu wel.

06

✨ Positief punt

Het moment op 21 april waarop de biker game eindelijk volledig stabiel draaide in React Native Web na dagen van debugging was echt een opluchting. Ik committe onmiddellijk, schreef even op wat ik gedaan had en voelde voor het eerst deze periode dat het project echt op het goede spoor zit. Met nog twee weken te gaan voelt dat als een heel goed moment om sterk af te ronden!
